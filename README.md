# Prototype PoC

The project is to create a core testing bed for key concepts we have developed around here.news. We will try to use this prototype to define some key functions of MVP development and understand the key UX concerns by using real data, and partially real users(internal team members) to showcase the potential of the contemplations we have been working on in the passing months. 

# Epic description

The whole epic is to enable internal users(with recognizable IDs), and the zombie accounts they can switch, to do simple posting, commenting, upvoting/downvoting jobs, with consuming their pre-allocated credits under each account. 

Two stories under this epic: 
1. Dev team will create such a prototype and improve its functions by integrating frontend and backend components(including prelinmary data structure), testing potential stacks to make the prototype viable
2. Content/Community team will keep using the system with existing content, and evaluate the functions to make sure their basic actions can be well executed, together with personal feedbacks. 

# Methodology

We will have the simplest UX design(by using Figma) referring to Product Definition Spec and the Epic description above, then transferring into code implementation, and deploy it on a remote server under domain name (0.here.news) to share with internal users only.
